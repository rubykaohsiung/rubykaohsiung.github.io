Ruby on Rails 指南指南

本指南記錄了在 Rails 指南上編寫 Ruby 的指南。本指南以一個優雅的迴圈方式跟隨自己，作為一個例子。

閱讀本指南後，您將瞭解：

  • 關於 Rails 文件中使用的約定。
  • 如何在本地生成指南。

1 降價

指南寫在 GitHub Flavored Markdown 中。有全面的 Markdown 文件，以及 備忘單。

2 序幕

每個指南都應該從頂部的動機文字開始（這是藍色區域的小介紹）。序言應該告訴讀者指南的內容，以及他們將學到什麼。例如，請參閱路由指南。

3 標題

每個指南的標題都使用h1 標題；指南部分使用h2 標題；小節使用h3 標題；等等。請注意，生成的 HTML 輸出將使用以<h2> 開頭的標題標籤。

Guide Title
===========

Section
-------

### Sub Section

寫標題時，除了介詞、連詞、內部冠詞和動詞“to be”的形式外，所有單詞都大寫：

#### Assertions and Testing Jobs inside Components
#### Middleware Stack is an Array
#### When are Objects Saved?

使用與常規文字相同的內聯格式：

##### The `:content_type` Option

4 連結到 API

指向 API (api.rubyonrails.org) 的連結由指南生成器按以下方式處理：

包含釋出標籤的連結保持不變。例如

https://api.rubyonrails.org/v5.0.1/classes/ActiveRecord/Attributes/ClassMethods.html

未修改。

請在發行說明中使用這些，因為無論生成的目標如何，它們都應該指向相應的版本。

如果連結不包含釋出標籤並且正在生成邊緣指南，則域將替換為 edgeapi.rubyonrails.org。例如，

https://api.rubyonrails.org/classes/ActionDispatch/Response.html

變成

https://edgeapi.rubyonrails.org/classes/ActionDispatch/Response.html

如果連結不包含釋出標籤並且正在生成釋出指南，則會注入 Rails 版本。例如，如果我們為 v5.1.0 生成指南，連結

https://api.rubyonrails.org/classes/ActionDispatch/Response.html

變成

https://api.rubyonrails.org/v5.1.0/classes/ActionDispatch/Response.html

請不要手動連結到edgeapi.rubyonrails.org。

6 HTML 指南

在生成指南之前，請確保您擁有最新版本的 Bundler 安裝在您的系統上。您可以找到最新的 Bundler 版本 此處。在撰寫本文時，它是 v1.17.1。

要安裝最新版本的 Bundler，請執行 gem install bundler。

6.1 一代

要生成所有指南，只需cd 進入guides 目錄，執行bundle install，然後執行：

$ bundle exec rake guides:generate

或者

$ bundle exec rake guides:generate:html

生成的 HTML 檔案可以在 ./output 目錄中找到。

要處理 my_guide.md 而沒有其他任何事情，請使用 ONLY 環境變數：

$ touch my_guide.md
$ bundle exec rake guides:generate ONLY=my_guide

預設情況下，沒有被修改的指南是不被處理的，所以在實踐中很少需要ONLY。

要強制處理所有指南，請傳遞ALL=1。

如果要生成英語以外的語言的指南，可以將它們儲存在source（例如source/es）下的單獨目錄中，並使用GUIDES_LANGUAGE 環境變數：

$ bundle exec rake guides:generate GUIDES_LANGUAGE=es

如果要檢視可用於配置生成指令碼的所有環境變數，只需執行：

$ rake

6.2 驗證

請使用以下方法驗證生成的 HTML：

$ bundle exec rake guides:validate

特別是，標題會根據其內容生成 ID，這通常會導致重複。

7 Kindle 指南

7.1 一代

要為 Kindle 生成指南，請使用以下 rake 任務：

$ bundle exec rake guides:generate:kindle
